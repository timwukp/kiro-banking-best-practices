# Contributing Guidelines

## Before Committing

This repository contains best practices documentation for AWS Kiro in banking environments.

### What to Include
- Markdown documentation files (*.md)
- Code examples and configuration samples
- Architecture diagrams (text/ASCII format)

### What NOT to Include
- PDF files (regulatory documents)
- `.kiro/` folder (local configuration)
- Any personally identifiable information (PII)
- Customer names, email addresses, or contact information
- Proprietary or confidential information

## Pre-Commit Checklist

1. Run the pre-commit validation hook
2. Verify no PDF files are staged
3. Ensure no PII or customer data is present
4. Confirm all examples use placeholder data
